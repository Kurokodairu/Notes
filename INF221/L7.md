## Quicksort

### Algorithm
1. **Partition** array around a pivot (last element in Lomuto scheme)
2. **Recurse** on left and right subarrays

### Lomuto Partition

- Pivot = A[r]
- i tracks rightmost element \(\leq\) pivot
- j scans left to right
- After partition: elements \(\leq x\) are in A[p..q-1], elements \(> x\) in A[q+1..r]

### Loop Invariant (Partition correctness)

1. \(i < j\)
2. \(A[p..i] \leq x\)
3. \(A[i+1..j-1] > x\)
4. \(A[r] = x\)

### Performance

|Case|Split|Recurrence|Runtime|
|---|---|---|---|
|**Worst**|\(n-1\) : \(0\)|\(T(n) = T(n-1) + \Theta(n)\)|\(\Theta(n^2)\)|
|**Best**|\(n/2\) : \(n/2\)|\(T(n) = 2T(n/2) + \Theta(n)\)|\(\Theta(n \lg n)\)|
|**Any fixed split** (e.g. 9:1)|—|—|\(O(n \lg n)\)|

[!tip] Even alternating good/bad splits gives \(O(n \lg n)\) — only consistently bad splits cause \(\Theta(n^2)\).

### Randomized Quicksort

- Pick pivot randomly → wraps Partition with a random swap
- Avoids worst case on sorted input

## Sorting in Linear Time

### Lower Bound for Comparison Sorts

- Any comparison sort is a **decision tree** of binary comparisons
- Tree needs \(\geq n!\) leaves → height \(\geq \lg(n!) = \Omega(n \lg n)\)
- **Conclusion:** No comparison sort can beat \(\Omega(n \lg n)\) in the worst case
- Merge sort & heapsort are **asymptotically optimal** at \(O(n \lg n)\)

### Counting Sort

- **Not** a comparison sort → lower bound doesn't apply
- **Assumption:** values are integers in \(\{0, 1, \dots, k\}\)
- **Steps:**
    1. Count occurrences of each value → C[i]
    2. Compute prefix sums on C
    3. Place elements into output B using C as position guide
- **Runtime:**\(\Theta(n + k)\) → \(\Theta(n)\) when \(k = O(n)\)
- **Stable sort** (preserves order of equal keys)
- **Limitation:** impractical when \(k \gg n\) (e.g. sorting strings)

## Medians & Order Statistics

### Definitions

- \(i^{\text{th}}\)**order statistic** = \(i^{\text{th}}\) smallest element
- **Minimum:**\(i = 1\), **Maximum:**\(i = n\)
- **Median:**\(i = \lceil n/2 \rceil\) (lower median)

### Finding Min/Max

- Simple scan: \(n - 1\) comparisons → \(\Theta(n)\)

### Randomized Select (find \(i^{\text{th}}\) smallest)

- Like quicksort but **recurse into only one side**
- Uses Randomized-Partition, then checks if pivot index = \(i\)

|Case|Runtime|
|---|---|
|**Worst**|\(\Theta(n^2)\)|
|**Average**|\(\Theta(n)\)|

[!info] Average case: recurrence \(T(n) = T(n/\beta) + \Theta(n)\) with \(\beta > 1\). Master theorem case 3 gives \(T(n) = \Theta(n)\). Key insight: only **one** subproblem (vs. two in quicksort).
