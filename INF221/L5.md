
## Randomized Algorithms

**Problem:** Deterministic algorithms can hit worst-case $\Theta(n)$ on sorted input.
**Solution:** Randomize input order → expected performance $\Theta(\ln n)$.

### Randomize-in-Place

```text
Randomize-in-place(A)
  n = A.length
  for i = 1 to n
    swap A[i] with A[Random(i, n)]
```

- `Random(i, n)` returns $k \in \{i, \dots, n\}$ uniformly
- **Runtime:** $\Theta(n)$ (n swaps, n random calls, each $\Theta(1)$)
- **Correctness (loop invariant):** Before iteration $i$, `A[1..i-1]` holds any $(i-1)$-permutation with probability $\frac{(n-i+1)!}{n!}$
  - **Init:** Empty subarray, trivially holds
  - **Maintenance:** Bayes' theorem + independence → probability becomes $\frac{(n-i)!}{n!}$
  - **Termination:** At $i = n+1$, each full permutation has probability $\frac{1}{n!}$ ✓

## Combinatorics Refresher

| Concept | Formula | Description |
|---|---|---|
| **Permutations** | $n!$ | All orderings of $n$ elements |
| **k-Permutations** | $\frac{n!}{(n-k)!}$ | Ordered sequences of $k$ from $n$ |
| **k-Combinations** | $\binom{n}{k} = \frac{n!}{k!(n-k)!}$ | Unordered subsets of size $k$ from $n$ |

## Generating Random Numbers

> *"Any one who considers arithmetical methods of producing random digits is, of course, in a state of sin."* — von Neumann

### Types of RNGs

| Type | Key Properties |
|---|---|
| **Physical** | True randomness (radioactive decay, shot noise); slow rate |
| **Pseudo-random (PRNG)** | Deterministic; fast; long period desired; no correlations |
| **Quasi-random** | Low-discrepancy sequences (e.g. Sobol); good for high-dim integration |
| **Cryptographic** | Unpredictable; supports skip-ahead; uses encryption (e.g. AES) |

### PRNG Framework (Recurrence-Based)

- **State:** $\Sigma$
- **Seed:** $\Sigma_0 = \mathcal{S}(S)$
- **Iterate:** $\Sigma_j = \mathcal{I}(\Sigma_{j-1})$
- **Draw:** $r_j = \mathcal{D}(\Sigma_j)$

### Key Generators

| Generator | Period $\rho$ | Notes |
|---|---|---|
| **LCG** (Lewis et al.) | $\approx 2 \times 10^9$ | $\Sigma_{j} = (a\Sigma_{j-1}+c) \bmod m$; $a=7^5,\, c=0,\, m=2^{31}-1$. Short period, fails advanced tests. Watch for overflow! |
| **MRG32k3a** (L'Ecuyer) | $\approx 2^{191} \approx 10^{57}$ | Two 53-bit state vars, 3 prior values. Passes TestU01. |
| **Mersenne Twister** | $2^{19937}-1 \approx 10^{6000}$ | Feedback shift register; $k=19937$ bit state. Python's `random` default. Passes almost all tests. |
| **PCG64** (O'Neill 2014) | Large | Fixes MT statistical issues; smaller state. NumPy ≥1.17 default. |
| **Cryptographic (AES)** | N/A | $C_j = E(K, S+j)$; no recurrence; unpredictable; trivial skip-ahead; slower (unless HW-accelerated). |

### LCG Python Implementation

```python
class LCG:
    a = 16807
    c = 0
    m = 2**31 - 1

    def __init__(self, seed):
        self.sigma = seed

    def draw(self):
        self.sigma = (self.a * self.sigma + self.c) % self.m
        return self.sigma
```

> **Rule of thumb:** Only use $\sqrt[3]{\rho}$ numbers from an LCG to avoid correlations ($\approx 1290$ for this LCG).

## Building `Random(0, b)` from `Random(0, 1)`

> Constraint: $b = 2^n - 1$

### ✅ Correct: Binary construction

```text
Random(0, b)
  r = 0
  n = lg(b+1)
  for k = 1..n
    r = r + RANDOM(0, 1) * 2^(k-1)
  return r
```

- Builds number bit-by-bit: $r = \sum_{k=1}^{n} X_k \cdot 2^{k-1}$
- **Expected value:** $E[r] = b/2$ ✓
- **Uniform:** Each of $2^n$ outcomes has probability $1/2^n = 1/(b+1)$ ✓ (bits are independent)

### ❌ Incorrect: Summation approach

```text
Random(0, b)
  r = 0
  for k = 1..b
    r = r + RANDOM(0, 1)
  return r
```

- Correct mean $E[r] = b/2$, but **not uniform**
- Produces a **binomial distribution** $P(X=k) = \binom{b}{k} \cdot \frac{1}{2^b}$
- Middle values are far more likely than extremes
