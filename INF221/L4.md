## 1. Asymptotic Notation

### Big-O (Upper Bound)

$$O(g(n)) = \{ f(n) \mid \exists\, c > 0,\, n_0 > 0 \text{ s.t. } 0 \le f(n) \le c \cdot g(n) \;\forall\, n \ge n_0 \}$$

- $f(n)$ grows **no faster** than $g(n)$
- Examples: $n \in O(n)$, $n \in O(n^2)$, $5n^2 + 2n - 10 \in O(n^2)$, $10^{-6}n^3 \notin O(n^2)$

### Big-Omega (Lower Bound)

$$\Omega(g(n)) = \{ f(n) \mid \exists\, c > 0,\, n_0 > 0 \text{ s.t. } 0 \le c \cdot g(n) \le f(n) \;\forall\, n \ge n_0 \}$$

- $f(n)$ grows **at least as fast** as $g(n)$
- Examples: $n^2 \in \Omega(n)$, $5n^2 + 2n - 10 \in \Omega(n^2)$, $10^6 n^2 \notin \Omega(n^3)$

### Big-Theta (Tight Bound)

$$\Theta(g(n)) = \{ f(n) \mid \exists\, c_1, c_2, n_0 > 0 \text{ s.t. } 0 \le c_1 g(n) \le f(n) \le c_2 g(n) \;\forall\, n \ge n_0 \}$$

- $f(n)$ grows **at the same rate** as $g(n)$
- $f(n) \in \Theta(g(n)) \iff f(n) \in O(g(n))$ **and** $f(n) \in \Omega(g(n))$
- Examples: $5n^2 + 2n - 10 \in \Theta(n^2)$, but $\notin \Theta(n)$ (too fast) and $\notin \Theta(n^3)$ (too slow)

## 2. Proving Dominance with Limits

> **"$g(n)$ dominates $f(n)$"** means $g$ grows faster, i.e. $f(n) = O(g(n))$

| Limit | Conclusion |
| --- | --- |
| $\lim_{n\to\infty} \frac{f(n)}{g(n)} = 0$ | $g$ dominates $f$ → $f = O(g)$ |
| $\lim_{n\to\infty} \frac{f(n)}{g(n)} = \infty$ | $f$ dominates $g$ → $f = \Omega(g)$ |
| $\lim_{n\to\infty} \frac{f(n)}{g(n)} = c$, where $0 < c < \infty$ | Neither dominates → $f = \Theta(g)$ |

> [!tip] Use **L'Hôpital's rule** when the limit is not obvious.

### Worked Examples

**Ex 1:** $f = n$, $g = n^2$ → $\lim \frac{n}{n^2} = \lim \frac{1}{n} = 0$ → $n = O(n^2)$ ✅

**Ex 2:** $f = n^3$, $g = n^2$ → $\lim \frac{n^3}{n^2} = \lim n = \infty$ → $n^3 = \Omega(n^2)$ ✅

**Ex 3:** $f = 2n^2$, $g = n^2$ → $\lim \frac{2n^2}{n^2} = 2$ → $2n^2 = \Theta(n^2)$ ✅

**Ex 4 (L'Hôpital):** $f = \ln n$, $g = \sqrt{n}$

$$\lim_{n\to\infty} \frac{\ln n}{\sqrt{n}} = \lim \frac{1/n}{1/(2\sqrt{n})} = \lim \frac{2\sqrt{n}}{n} = \lim \frac{2}{\sqrt{n}} = 0$$

→ $\ln n = O(\sqrt{n})$ ✅

> [!warning] Dominance ($\lim = 0$) implies $O$, but $O$ does **not** imply dominance. Every function is in its own Big-O: $f(n) = O(f(n))$.

## 3. Solving Recurrences

### Example: Prove by Induction

**Recurrence:** $T(n) = 2T(n/2) + n$, $T(1) = 1$
**Claimed solution:** $T(n) = n \lg n + n$

**Base case:** $T(1) = 1 \lg 1 + 1 = 1$ ✅

**Inductive step:** Assume $T(k) = k \lg k + k$ for all $k < n$.

$$T(n) = 2T(n/2) + n = 2\left(\frac{n}{2}\lg\frac{n}{2} + \frac{n}{2}\right) + n$$

$$= n \lg\frac{n}{2} + 2n = n(\lg n - \lg 2) + 2n = n\lg n - n + 2n = n\lg n + n \quad \checkmark$$

---

### Master Theorem

For recurrences: $T(n) = aT(n/b) + f(n)$ where $a \ge 1$, $b > 1$, $f(n) > 0$

**Key value:** $n^{\log_b a}$ — compare $f(n)$ against this.

| Case | Condition | Solution | Intuition |
| --- | --- | --- | --- |
| **1** | $f(n) = O(n^{\log_b a - \epsilon})$ for some $\epsilon > 0$ | $T(n) = \Theta(n^{\log_b a})$ | Leaves dominate |
| **2** | $f(n) = \Theta(n^{\log_b a})$ | $T(n) = \Theta(n^{\log_b a} \lg n)$ | Equal cost per level |
| **3** | $f(n) = \Omega(n^{\log_b a + \epsilon})$ for some $\epsilon > 0$ **AND** $af(n/b) \le cf(n)$ for $c < 1$ | $T(n) = \Theta(f(n))$ | Root dominates |

> [!important] **Case 3 requires the regularity condition!** Always check $af(n/b) \le cf(n)$ for some $c < 1$.
> Regularity is always satisfied when $f(n) = n^k$ (polynomial).

### Master Theorem Examples

| Recurrence | $a, b$ | $n^{\log_b a}$ | vs $f(n)$ | Case | Solution |
| --- | --- | --- | --- | --- | --- |
| $5T(n/2) + \Theta(n^2)$ | $5, 2$ | $n^{\log_2 5} \approx n^{2.32}$ | $n^2$ smaller | 1 | $\Theta(n^{\lg 5})$ |
| $27T(n/3) + \Theta(n^3)$ | $27, 3$ | $n^{\log_3 27} = n^3$ | equal | 2 | $\Theta(n^3 \lg n)$ |
| $5T(n/2) + \Theta(n^3)$ | $5, 2$ | $n^{2.32}$ | $n^3$ larger | 3 | $\Theta(n^3)$ |

**Regularity check for Case 3 above:** $af(n/b) = 5(n/2)^3 = 5n^3/8 \le cn^3$ → $c = 5/8 < 1$ ✅

### When Master Theorem Fails

1. **Non-constant $a$:** e.g. $T(n) = 2^n T(n/4) + \Theta(1)$ — $a = 2^n$ depends on $n$ ❌
2. **Regularity fails:** e.g. $T(n) = 2T(n/2) + n^2(2 - \cos n)$
   - $n^{\log_b a} = n$, $f(n) \ge n^2$ → seems like Case 3
   - But near $n \approx 2(2k+1)\pi$: $\cos n \approx 1$ so $f(n) \approx n^2$, while $af(n/b) \approx \frac{3}{2}n^2$
   - Need $c \ge 3/2$ but require $c < 1$ — **contradiction** ❌

## 4. Probabilistic Analysis — The Technology (IDE) Problem

### Setup

- Evaluate $n$ IDEs; switch when a better one is found
- Evaluate cost: $c_E$, switch cost: $c_S$ (where $c_S > c_E$)
- Total cost: $C(n, m) = nc_E + mc_S$ where $m$ = number of switches

| Scenario | Switches $m$ | Cost |
| --- | --- | --- |
| **Worst** | $m = n$ (sorted worst→best) | $nc_E + nc_S \approx nc_S$ |
| **Best** | $m = 1$ | $nc_E + c_S \approx nc_E$ |
| **Average** | $m = \ln n + O(1)$ | $nc_E + c_S \ln n \approx c_S \ln n$ |

### Deriving the Average

Assume IDEs arrive in **random order**. We switch to IDE $i$ only if it's the best among the first $i$ tested.

$$\Pr\{\text{switch to IDE } i\} = \frac{1}{i}$$

$$E[X] = \sum_{i=1}^{n} \frac{1}{i} = H_n = \ln n + \gamma + O(1/n)$$

where $H_n$ is the **$n$-th Harmonic number** and $\gamma \approx 0.57722$ is the **Euler–Mascheroni constant**.

> [!note] Average switches grow as $\Theta(\ln n)$ — **much** better than worst case $\Theta(n)$.

## 5. Indicator Random Variables

### Definition

$$I\{A\} = \begin{cases} 1 & \text{if event } A \text{ occurs} \\ 0 & \text{otherwise} \end{cases}$$

### Key Lemma

$$E[X_A] = E[I\{A\}] = \Pr\{A\}$$

**Proof:**

$$E[I\{A\}] = 1 \cdot \Pr\{A\} + 0 \cdot \Pr\{\bar{A}\} = \Pr\{A\}$$

### Why It's Powerful

**Linearity of expectation** works even for **dependent** variables:

$$E\left[\sum_{i=1}^{n} X_i\right] = \sum_{i=1}^{n} E[X_i]$$

### Coin Flip Example

Expected heads in $n$ fair coin flips:

$$X_i = I\{i\text{-th flip is H}\}, \quad E[X_i] = 1/2$$

$$E[X] = \sum_{i=1}^{n} E[X_i] = \sum_{i=1}^{n} \frac{1}{2} = \frac{n}{2}$$

## 6. Randomized Algorithms

### Motivation

- Worst case for IDE problem: inputs sorted worst→best
- **Solution:** randomize input order → worst case can't be forced
- Never worse than $\Theta(\ln n)$ expected switches

### Fisher-Yates Shuffle (Randomize-in-Place)

```text
Randomize-in-place(A)
    n = A.length
    for i = 1 to n
        swap A[i] with A[Random(i, n)]
```

- **Runtime:** $\Theta(n)$ ($n$ swaps, $n$ random numbers)
- **Produces uniform random permutation** — each of $n!$ permutations has probability $1/n!$

### Proof of Correctness (Loop Invariant)

**Invariant:** Before iteration $i$, subarray $A[1..i-1]$ contains any $(i-1)$-permutation of $n$ elements with probability $\frac{(n-i+1)!}{n!}$

- **Init ($i=1$):** Empty subarray, $1$ zero-permutation with prob $n!/n! = 1$ ✅
- **Maintenance:** Using Bayes' theorem:
  - $E_1$: first $i-1$ elements correctly placed → prob $\frac{(n-i+1)!}{n!}$ (induction)
  - $E_2$: $x_i$ placed at position $i$ → prob $\frac{1}{n-i+1}$ (uniform random from remaining)
  - $\Pr\{E_2 | E_1\} = \Pr\{E_2\}$ (independent — selection is from $A[i..n]$)
  - $\Pr\{E_1 \cap E_2\} = \frac{1}{n-i+1} \cdot \frac{(n-i+1)!}{n!} = \frac{(n-i)!}{n!}$ ✅
- **Termination ($i=n+1$):** $A[1..n]$ holds any $n$-permutation with prob $\frac{0!}{n!} = \frac{1}{n!}$ ✅

### Combinatorics Formulas

| Concept | Formula |
| --- | --- |
| Permutations of $n$ elements | $n!$ |
| $k$-permutations (ordered, no replacement) | $\frac{n!}{(n-k)!}$ |
| $k$-combinations (unordered, no replacement) | $\binom{n}{k} = \frac{n!}{k!(n-k)!}$ |

## 7. Random Number Generators

### Types Overview

| Type | How it works | Key property |
| --- | --- | --- |
| **Physical** | Radioactive decay, shot noise | True randomness, but slow |
| **Pseudo-random (PRNG)** | Deterministic algorithm from seed | Fast, reproducible, periodic |
| **Quasi-random** | Low-discrepancy sequences (e.g. Sobol) | Space-filling, not "random" |
| **Cryptographic** | Encryption-based, no recurrence | Unpredictable, skip-ahead |

### PRNG Framework

$$\Sigma_0 = \mathcal{S}(S) \quad \text{(seed)}$$

$$\Sigma_j = \mathcal{I}(\Sigma_{j-1}) \quad \text{(iterate)}$$

$$r_j = \mathcal{D}(\Sigma_j) \quad \text{(draw)}$$

### Linear Congruential Generator (LCG)

$$\Sigma_j = (a\Sigma_{j-1} + c) \bmod m$$

Popular params (Lewis et al, 1969): $a = 7^5 = 16807$, $c = 0$, $m = 2^{31} - 1$

- Max number: $M = 2^{31} - 2 \approx 2.1 \times 10^9$
- Period $\rho = M$ (maximal)
- **Weakness:** short period, fails advanced tests, use only $\sqrt[3]{\rho} \approx 1290$ numbers safely
- **Beware** integer overflow in 32-bit implementations!

### MRG32k3a (L'Ecuyer, 1999)

- Two state variables $\Sigma_1, \Sigma_2$ (53-bit), each depending on 3 previous values
- Period $\rho \approx 2^{191} \approx 10^{57}$
- Passes all TestU01 tests

### Mersenne Twister (1998)

- Feedback shift register, arithmetic mod 2 (bit operations)
- State: vector of $k = 19937$ bits
- $\Sigma_j = \mathbf{A}\Sigma_{j-1}$, draw via output matrix $\mathbf{B}$ ($w = 53$ bits)
- Period: $2^{19937} - 1 \approx 10^{6000}$
- Python's `random` module default
- Passes almost all tests

### Permuted Congruential Generators (PCG, O'Neill 2014)

- Fixes statistical issues in Mersenne Twister
- Much smaller state
- **NumPy default** since v1.17

### Cryptographic Generators

$$C_j = E(K, S + j)$$

- $K$ = secret key, $S$ = seed, $E$ = encryption (e.g. AES)
- **No recurrence** — $C_j$ independent of $C_{j-1}$
- **Advantages:** unpredictable (key secret), trivial skip-ahead
- **Disadvantage:** slower (mitigated by hardware AES support)

## 8. Building `Random(0, b)` from `Random(0, 1)`

Where $b = 2^n - 1$.

### ✅ Correct: Build as Binary Number

```text
r = 0
n = lg(b + 1)
for k = 1..n
    r = r + RANDOM(0, 1) * 2^(k-1)
return r
```

**Proof of uniform distribution:** Any value $m = \sum_{k=1}^{n} m^{(k)} 2^{k-1}$ has probability:

$$\Pr\{r = m\} = \prod_{k=1}^{n} \Pr\{X_k = m^{(k)}\} = \frac{1}{2^n} = \frac{1}{b+1}$$

(Each bit independent, each has prob $1/2$.)

**Expected value:** $E[r] = \sum_{k=1}^{n} \frac{1}{2} \cdot 2^{k-1} = \frac{2^n - 1}{2} = \frac{b}{2}$ ✅

### ❌ Incorrect: Summing Bits

```text
r = 0
for k = 1..b
    r = r + RANDOM(0, 1)
return r
```

- Correct mean ($b/2$) but **binomial distribution**, not uniform
- For $b = 2$: $\Pr\{r=0\} = 1/4$, $\Pr\{r=1\} = 1/2$, $\Pr\{r=2\} = 1/4$ → **not uniform** ❌
- General distribution: $P(X = k) = \binom{b}{k} \cdot \frac{1}{2^b}$
